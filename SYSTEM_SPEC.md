# パーソナライズニュース配信Bot システム仕様書

## 1. プロジェクト概要
ユーザーが登録したキーワードに基づき、Google Newsから最新ニュースを収集・解析・要約し、LINEでパーソナライズされたニュース配信を行うシステム。

## 2. システムアーキテクチャ
- **Backend**: Flask (Renderデプロイ)
- **Database**: Google Sheets (ユーザー管理、キーワード重み、送信履歴)
- **AI**: Gemini 3.0 Flash preview (ニュース要約)
- **Interface**: LINE Messaging API (Flex Message / Carousel, Postback Actions)
- **Automation**: GitHub Actions (定期実行スクリプト)

## 3. 実装済み機能詳細

### ① ニュース取得・加工ロジック
- **RSS収集 (`rss.py`)**: Google News RSSを使用。キーワード検索による記事取得。
- **類義語展開 (`expand_keywords.py`)**: 登録キーワードから類義語（例：AI → 人工知能, LLM）を自動展開し、検索母数を拡大。
- **重複排除 (`dedup.py`)**: タイトルの正規化（記号・大文字小文字無視）による同一内容記事の排除。PR・広告記事のフィルタリング。

### ② パーソナライズ・スコアリング (`scoring.py`)
記事ごとに以下の計算式でランク付けを行い、上位記事のみを配信。
`Score = (キーワード重みスコア * 0.4) + (カテゴリスコア * 0.3) + (鮮度スコア * 0.2) + (ソース信頼度スコア * 0.1)`
- **キーワード重み**: スプレッドシートに保存されたキーワードごとの `weight` を反映。
- **鮮度**: 24時間以内(1.0)、48時間以内(0.7)、それ以降(0.3)で減衰。
- **信頼度**: NHK、日経、ロイター等の主要メディアに固定加点。

### ③ AI要約 (`summarize_gemini.py`)
- **モデル**: `gemini-1.5-flash` (高速・安定版)
- **プロンプト**: 結論を冒頭に配置し、3つのポイントで構成される100〜150文字の要約を生成。
- **エラー耐性**: 指数バックオフによる最大3回のリトライ処理（503負荷エラー対策）。

### ④ LINE UI・インタラクション (`line_format.py`, `line_webhook.py`)
- **Flex Message**: タイトル、要約、リンクボタンを備えたカード型UI。
- **カルーセル**: 最大10件の記事を横スワイプで閲覧可能。
- **フィードバックボタン**:
    - 👍 **いいね**: 記事に含まれるキーワードの重みを `+0.2` 加算。
        - ログ: `article_log` に `like` アクションを記録。
    - 👎 **興味なし**: 記事に含まれるキーワードの重みを `-0.3` 減算。
        - ログ: `article_log` に `dislike` アクションを記録。
    - **もっと**: 非同期処理（Threading）により、即座に追加ニュース5件を生成・配信。
    - **続きを読む**: 記事URLへのリンク。
        - ログ: `article_log` に `click` アクションを記録。
- **「傾向」コマンド**: ユーザーの行動ログ（直近7日間）から興味傾向を分析し、要約してLINEで表示。
- **プロファイルフィードバックUI**: (未実装) ユーザー傾向表示時に「👍このままでOK」「👎調整したい」ボタンを提示。

### ⑤ 運用・非機能要件
- **非同期実行**: Webhook内の重い処理を別スレッドで実行し、LINE側のタイムアウト(30s)を回避。
- **認証管理**: サービスアカウントJSONのBase64エンコードによる環境変数管理。
- **二重送信防止**: `history` シートへのURL記録による既読管理。
- **記事特徴抽出 (`feature_extractor.py`)**: 記事タイトル・要約からルールベースでキーワードとカテゴリを抽出。
- **ユーザープロファイル生成 (`profile.py`)**: `article_log` からユーザーのキーワード・カテゴリごとの反応数を集計。

## 4. データ構造 (Google Sheets)

### `sheet1` (ユーザー名簿)
- `LINE_USER_ID`: ユーザー識別子
- `KEYWORDS`: カンマ区切りのキーワード文字列
    - **備考**: `keywords` シートへの移行を推奨。

### `article_log` (ユーザー行動ログ)
- `user_id`: ユーザー識別子
- `article_id`: 記事URL (ユニークID)
- `keyword`: 記事にマッチしたキーワード (カンマ区切り)
- `category`: 記事のカテゴリ
- `action`: `send`, `click`, `like`, `dislike`
- `timestamp`: 行動日時 (ISO 8601)

### `keywords` (重み管理)
- `user_id`: ユーザーID
- `keyword`: 単一キーワード
- `weight`: 興味関心度（デフォルト 1.0）

### `history` (送信履歴)
- `user_id`: ユーザーID
- `article_id`: 記事URL (ユニークID)

### `user_profile` (ユーザープロファイルキャッシュ) - (未実装)
- `user_id`: ユーザー識別子
- `keyword`: キーワード
- `count`: 反応数
- `category`: カテゴリ
- `count`: 反応数
